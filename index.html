What is OpenClaw?

Frequently Asked Questions

What is OpenClaw? ‚ñº

OpenClaw is an open-source, locally-running AI assistant that operates directly on your machine. Unlike cloud-based AI services like ChatGPT or Claude's web interface, OpenClaw runs on your hardware, giving you complete control over your data and capabilities.

Previously known as Moltbot and originally Clawdbot, it supports multiple AI models (Claude, GPT-4, Gemini, local LLMs), browser automation, device integration, and extensibility through 700+ community skills.

What are OpenClaw Skills? ‚ñº

OpenClaw Skills are modular extensions that teach your AI agent how to use specific tools and services. Each skill is a directory containing a SKILL.md file with YAML frontmatter and instructions.

Skills follow the AgentSkills convention developed by Anthropic. They can integrate with APIs (GitHub, Slack, Supabase), automate browsers (Playwright, Puppeteer), manage infrastructure (Kubernetes, Docker), and much more.

How do I install OpenClaw skills? ‚ñº

The easiest way is using the ClawdHub CLI:

npx clawdhub@latest install <skill-name>

Alternatively, manually copy the skill folder to:

  • Global: ~/.openclaw/skills/
  • Workspace: <project>/skills/

Workspace skills take precedence over global skills.

Are OpenClaw skills free? ‚ñº

Yes! All skills in the OpenClaw Skills Directory are free and open-source under permissive licenses. You can use, modify, and redistribute them.

However, some skills require API keys for external services (like OpenAI, Brave Search, Supabase, etc.) which may have their own pricing tiers. Many services offer generous free tiers.

How is OpenClaw different from ChatGPT or Claude? ‚ñº

Key differences:

  • Local execution: OpenClaw runs on your machine with full system access
  • Multi-model: Use Claude, GPT-4, Gemini, or local LLMs - your choice
  • Browser control: Automate Chrome/Chromium with snapshots and actions
  • Device integration: Connect to mobile devices for camera, screen, location
  • Extensible: 700+ community skills vs limited plugins
  • Open source: Full transparency and customization
How do I create my own skill? ‚ñº

Create a new folder with a SKILL.md file containing YAML frontmatter:

---
name: my-skill
description: What this skill does
---

Instructions for the AI agent on how to use this skill...

Place it in ~/.openclaw/skills/my-skill/ and it will be loaded on your next session. See the full documentation for advanced options.

What API keys do I need? ‚ñº

It depends on which skills you use. Common API keys:

  • AI Models: ANTHROPIC_API_KEY, OPENAI_API_KEY, GOOGLE_API_KEY
  • Search: BRAVE_API_KEY, TAVILY_API_KEY, EXA_API_KEY
  • Services: GITHUB_TOKEN, SLACK_TOKEN, SUPABASE_KEY

Add keys to ~/.openclaw/openclaw.json under skills.entries.<skill-name>.apiKey
